# 서비스 문서화 생성 스킬

## 개요

서비스 컨셉을 입력받아 프로젝트 전체 문서를 체계적으로 생성하는 스킬입니다.

## 트리거

사용자가 다음과 같이 요청할 때 이 스킬을 사용합니다:
- "서비스 문서화 해줘"
- "프로젝트 문서 생성해줘"
- "이 컨셉으로 문서 만들어줘"
- 새 프로젝트 초기 설정 요청

## 입력 수집

스킬 실행 전 사용자에게 다음 정보를 수집합니다:

### 필수 정보
1. **프로젝트명**: 내부 코드명 (예: Project Lattice)
2. **서비스명**: 사용자에게 보이는 브랜드명 (예: 캘리런)
3. **한 줄 설명**: 서비스의 핵심 가치 요약
4. **타겟 사용자**: 1차 타겟과 확장 타겟
5. **핵심 기능**: 3-5개의 핵심 기능
6. **기술 스택**: Frontend, Backend, DB, 인프라 등

### 선택 정보
- 비즈니스 모델 (Freemium, 구독 등)
- MVP 범위
- 특수 요구사항 (AI, 실시간, 오프라인 등)

## 생성할 문서 구조

```
{project}/
├── CLAUDE.md                    # AI 코딩 가이드
├── README.md                    # 프로젝트 소개
├── .cursorrules                 # AI 코딩 규칙
└── docs/
    ├── 01-prd.md               # 제품 요구사항 정의서
    ├── 02-architecture.md      # 시스템 아키텍처
    ├── 03-{domain}-strategy.md # 도메인별 전략 (AI, 콘텐츠 등)
    ├── 04-api-spec.md          # API 명세
    ├── 05-roadmap.md           # 개발 로드맵
    ├── 06-dev-guide.md         # 개발 가이드
    ├── 07-mvp-checklist.md     # MVP 구현 체크리스트
    └── policies/               # 정책 문서
        └── {policy-name}.md
```

## 문서별 생성 가이드

### 1. CLAUDE.md

AI 코딩 어시스턴트가 참조하는 프로젝트 가이드 파일입니다.

필수 섹션:
- 프로젝트 개요 (프로젝트명, 서비스명, 목적)
- 핵심 기능 요약
- 기술 스택
- 프로젝트 구조 (모노레포 구조)
- 핵심 상수 정의
- 문서 참조 테이블
- 바이브 코딩 가이드

### 2. docs/01-prd.md (제품 요구사항)

필수 섹션:
1. 제품 개요 (프로젝트명, 서비스명, 한 줄 정의, USP)
2. 타겟 사용자 (1차 타겟, 확장 타겟, 사용자 문제)
3. 솔루션 정의
4. 사용자 핵심 플로우 (단계별)
5. 기능 요구사항 (카테고리별)
6. 비기능 요구사항 (성능, 보안, 확장성)
7. MVP 범위 (포함/제외)
8. KPI (핵심 지표)
9. 수익 모델

### 3. docs/02-architecture.md (시스템 아키텍처)

필수 섹션:
1. 모노레포 구조 (디렉토리 트리)
2. 기술 스택 (플랫폼별 테이블)
3. 데이터 흐름 (ASCII 다이어그램)
4. 데이터 모델 (SQLModel/Prisma 코드)
5. ERD (텍스트 표현)
6. 핵심 설계 포인트
7. 워크플로우 설계 (있는 경우)
8. 확장 전략

### 4. docs/03-{domain}-strategy.md (도메인 전략)

서비스 특성에 따라 도메인별 전략 문서 생성:
- AI 서비스: `03-ai-strategy.md` (생성 전략, 품질 검증, 프롬프트 관리)
- 콘텐츠 서비스: `03-content-strategy.md`
- 커머스: `03-commerce-strategy.md`
- 등등

### 5. docs/04-api-spec.md (API 명세)

필수 섹션:
1. 개요 (Base URL, 인증 방식)
2. 엔드포인트 목록 (카테고리별 테이블)
3. 요청/응답 스키마 (Pydantic/TypeScript)
4. 에러 모델 (코드, HTTP 상태, 설명)
5. 상세 API 명세 (주요 API)
6. 웹훅 (내부용)

### 6. docs/05-roadmap.md (개발 로드맵)

필수 섹션:
1. 릴리즈 단계 (R0 MVP, R1, R2...)
2. 문서화 로드맵 (Phase별)
3. 핵심 원칙 (서비스 원칙, 기술 상수)
4. 리스크 및 완화 전략
5. 문서 디렉토리 구조 (목표)
6. 다음 단계

### 7. docs/06-dev-guide.md (개발 가이드)

필수 섹션:
1. 개발 환경 (필수 도구, 초기 설정, 개발 서버)
2. 프로젝트 구조 (상세 디렉토리)
3. 코딩 컨벤션 (언어별)
4. API 개발 패턴 (라우터, 서비스 레이어, 에러 처리)
5. Frontend 개발 패턴
6. 테스트 (Backend, Frontend)
7. 배포 (환경 변수, Docker)
8. 바이브 코딩 팁

### 8. docs/07-mvp-checklist.md (MVP 체크리스트)

필수 섹션:
1. 개요
2. Phase별 체크리스트 (체크박스 형태)
   - Phase 1: 프로젝트 초기화
   - Phase 2: 인증 시스템
   - Phase 3~N: 핵심 기능별
   - Phase N+1: 통합 및 마무리
3. 구현 우선순위 요약 (테이블)
4. 바이브 코딩 진행 방식

### 9. docs/policies/*.md (정책 문서)

서비스 특성에 따른 정책 문서:
- 핵심 비즈니스 로직을 정책으로 문서화
- 상수값, 트리거 조건, 처리 방식 명시
- UX 고려사항 포함

### 10. .cursorrules (AI 코딩 규칙)

필수 섹션:
1. 프로젝트 개요
2. 문서 참조 목록
3. 기술 스택 (플랫폼별)
4. 코딩 컨벤션 (언어별 코드 예시)
5. API 패턴
6. Frontend 패턴 (플랫폼별)
7. 에러 처리
8. 테스트
9. 핵심 상수
10. Git 컨벤션
11. 작업 요청 시 참고사항

## 실행 순서

1. **정보 수집**: 사용자에게 필수/선택 정보 질문
2. **docs/ 폴더 생성**
3. **핵심 문서 생성** (순서대로):
   - 01-prd.md
   - 02-architecture.md
   - 03-{domain}-strategy.md
   - 04-api-spec.md
   - 05-roadmap.md
   - 06-dev-guide.md
   - 07-mvp-checklist.md
4. **정책 문서 생성**: docs/policies/
5. **루트 파일 생성**:
   - CLAUDE.md
   - README.md
   - .cursorrules
6. **결과 요약**: 생성된 문서 목록과 다음 단계 안내

## 품질 체크리스트

생성된 문서가 다음을 만족하는지 확인:
- [ ] 모든 문서가 한국어로 작성됨
- [ ] 파일명이 숫자 접두사 + 소문자 + 하이픈 규칙을 따름
- [ ] 문서 간 중복 내용 없음 (상호 참조 사용)
- [ ] 코드 예시가 기술 스택에 맞게 작성됨
- [ ] 체크리스트가 실행 가능한 수준으로 구체적임
- [ ] 핵심 상수가 명시되어 있음
- [ ] 바이브 코딩을 위한 가이드가 포함됨

## 주의사항

- 서비스 특성에 맞게 도메인 전략 문서명 변경
- 기술 스택에 따라 코드 예시 조정
- 정책 문서는 서비스 핵심 비즈니스 로직에 맞게 생성
- 이미 존재하는 문서가 있으면 덮어쓰기 전 확인
