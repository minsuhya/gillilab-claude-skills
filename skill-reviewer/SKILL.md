# Skill Reviewer

다른 스킬의 품질과 효과성을 체계적으로 검토하는 메타 스킬.

## 검토 워크플로우

### 1. 대상 스킬 로드

사용자가 스킬 이름을 지정하면 해당 스킬 디렉터리를 탐색한다.

```
~/.claude/skills/{skill-name}/
├── SKILL.md
├── scripts/
├── references/
└── assets/
```

스킬 이름이 없으면 `~/.claude/skills/` 하위 전체 스킬 목록을 보여주고 선택하게 한다.

### 2. 검토 항목

4개 영역을 순서대로 평가한다. 각 항목에 심각도를 부여한다.

#### A. 효과성 평가 (Effectiveness)

| 점검 항목 | 기준 |
|-----------|------|
| description 트리거 범위 | 의도된 사용 시나리오를 모두 포괄하는가? |
| description 과잉 트리거 | 관련 없는 요청에도 트리거될 수 있는가? |
| 명령어 명확성 | Claude가 모호함 없이 실행할 수 있는가? |
| 컨텍스트 효율성 | SKILL.md가 500줄 이내인가? 불필요한 내용이 있는가? |
| Progressive Disclosure | 상세 내용이 references로 적절히 분리되었는가? |

#### B. 명령어 개선 (Instructions)

| 점검 항목 | 기준 |
|-----------|------|
| 동사형 사용 | 명령형/부정사형으로 작성되었는가? |
| 구체성 | 추상적 지시 대신 구체적 행동이 명시되어 있는가? |
| 예시 포함 | 핵심 워크플로우에 코드/명령 예시가 있는가? |
| 자유도 적절성 | 작업 위험도에 맞는 자유도를 부여하는가? |

#### C. 엣지 케이스 식별 (Edge Cases)

누락 가능한 시나리오를 탐색한다:

- 빈 입력, 잘못된 형식의 입력
- 대용량 파일 또는 다수 파일 처리
- 권한 부족, 네트워크 미연결 등 환경 제약
- 의존 도구(pip 패키지, CLI 등) 미설치 상태
- 다국어 또는 특수문자 포함 입력
- 스킬의 도메인 특성에 따른 고유 엣지 케이스

#### D. 구조 변경 권장 (Structure)

| 점검 항목 | 기준 |
|-----------|------|
| 디렉터리 구성 | scripts/references/assets 분리가 적절한가? |
| 파일 크기 | 단일 파일이 과도하게 크지 않은가? |
| 불필요 파일 | README, CHANGELOG 등 불필요 파일이 포함되어 있는가? |
| 리소스 참조 | SKILL.md에서 번들 리소스를 명확히 참조하는가? |
| frontmatter | name과 description만 포함하는가? |

### 3. 결과 리포트 형식

```markdown
# 스킬 검토: {skill-name}

## 요약
- 전체 등급: A/B/C/D/F
- Critical: N건 | Major: N건 | Minor: N건 | Suggestion: N건

## Critical (즉시 수정 필요)
- [C1] {영역}: {내용} → 권장 수정안

## Major (수정 권장)
- [M1] {영역}: {내용} → 권장 수정안

## Minor (개선하면 좋음)
- [m1] {영역}: {내용} → 권장 수정안

## Suggestion (선택적 제안)
- [S1] {영역}: {내용}

## 강점
- {잘 된 부분 나열}
```

### 4. 심각도 기준

| 등급 | 의미 | 예시 |
|------|------|------|
| **Critical** | 스킬이 동작하지 않거나 잘못된 결과를 낼 수 있음 | frontmatter 누락, description 빈값 |
| **Major** | 효과성이 크게 저하됨 | 트리거 범위 부족, 핵심 워크플로우 누락 |
| **Minor** | 품질 개선 여지가 있음 | 컨텍스트 비효율, 예시 부족 |
| **Suggestion** | 선택적 개선 아이디어 | 대안 구조, 추가 기능 |

### 5. 전체 등급 기준

| 등급 | 조건 |
|------|------|
| **A** | Critical 0, Major 0 |
| **B** | Critical 0, Major 1-2 |
| **C** | Critical 0, Major 3+ |
| **D** | Critical 1 |
| **F** | Critical 2+ |

## 일괄 검토 모드

"모든 스킬 검토" 요청 시, `~/.claude/skills/` 내 전체 스킬을 순회하며 요약 테이블을 출력한다:

```markdown
| 스킬 | 등급 | Critical | Major | Minor | 핵심 이슈 |
|------|------|----------|-------|-------|-----------|
| skill-a | A | 0 | 0 | 2 | - |
| skill-b | D | 1 | 1 | 0 | frontmatter 누락 |
```

## 주의사항

- 검토 대상에서 `skill-reviewer` 자신은 제외한다.
- 스킬 파일을 직접 수정하지 않는다. 권장 수정안만 제시한다.
- 사용자가 수정 적용을 요청하면 공통 이슈를 먼저 일괄 처리한 후 개별 이슈 순으로 진행한다. 각 수정 후 quick_validate.py로 검증한다.
- 일괄 검토 시 알파벳순으로 순회한다.
