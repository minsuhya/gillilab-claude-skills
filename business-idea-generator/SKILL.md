# Business Idea Generator 스킬

## 개요

비즈니스 트렌드 분석부터 실행 가능한 비즈니스 모델까지, **6단계 체계적 워크플로우**로 창업 아이디어를 발굴하고 검증합니다.

```
Phase 1: 컨텍스트 수집      → 사용자 배경 & 목표 파악
Phase 2: 트렌드 리서치      → AI/Tech, 시장, SNS, 경쟁사 분석
Phase 3: 아이디어 도출      → 10개+ 후보 → 평가 매트릭스로 TOP5 선정
Phase 4: 타당성 분석        → 시장성/구현가능성/수익성 심층 분석
Phase 5: 비즈니스 모델      → BMC 9블록 + 재무 예측
Phase 6: 실행 계획          → 로드맵 + 즉시 실행 액션 플랜
```

---

## 실행 방법

```bash
/business-idea-generator
```

인자 옵션:
```bash
/business-idea-generator --domain "이커머스"          # 특정 도메인 지정
/business-idea-generator --skip-interview              # 인터뷰 생략 (기본값으로 진행)
/business-idea-generator --top 3                       # TOP3만 분석 (기본값: TOP5)
/business-idea-generator --output-dir "my-ideas"       # 출력 디렉토리 지정
```

## Phase 0: 초기 설정 및 컨텍스트 수집

### 0.1 출력 디렉토리 생성

기본값으로 `{project-root}/business-ideas/` 디렉토리를 생성합니다.
`--output-dir` 인자가 있으면 해당 경로 사용.

```
{output-dir}/
├── README.md
├── 01-research/
├── 02-ideas/
├── 03-business-models/
├── 04-roadmap/
└── 05-next-steps/
```

### 0.2 사용자 컨텍스트 인터뷰 (AskUserQuestion 사용)

`--skip-interview` 옵션이 없을 경우 다음 질문을 통해 맞춤형 분석 수행:

**질문 1**: 현재 보유 기술/도메인 전문성
- 옵션: IT/소프트웨어, 이커머스/유통, 제조/오프라인, 서비스업, 기타

**질문 2**: 목표 시장
- 옵션: B2C (일반 소비자), B2B (기업), B2B2C (양면 플랫폼), 글로벌

**질문 3**: 초기 투자 가능 규모
- 옵션: 부트스트랩 (<1억), 소규모 (1-5억), 시드 투자 필요 (5-30억), 시리즈A 타겟

**질문 4**: 팀 구성 현황
- 옵션: 1인 창업자, 공동창업자 있음, 기존 팀 있음

수집된 컨텍스트는 모든 후속 분석에 반영됩니다.

## Phase 2: 아이디어 도출 및 TOP5 선정

### 2.1 아이디어 브레인스토밍 (10-15개 후보)

Phase 1 리서치 결과를 기반으로, 다음 프레임워크로 아이디어를 생성합니다:

**생성 프레임워크**:
1. **Pain Point → Solution**: 발견된 Pain Point를 직접 해결하는 아이디어
2. **Technology Push**: 새 기술(AI, API)로 기존 문제를 더 잘 해결
3. **Market Gap**: 경쟁사가 커버하지 못한 니치 시장
4. **Platform Extension**: 기존 플랫폼 위에 올리는 레이어 비즈니스
5. **Trend Riding**: 급성장 트렌드에 올라타는 아이디어

### 2.2 평가 매트릭스 → `02-ideas/top5-evaluation-matrix.md`

각 아이디어를 **5개 기준 × 5점 만점 = 25점** 으로 평가:

| 기준 | 배점 | 평가 방법 |
|------|------|-----------|
| **시장 규모** | /5 | TAM > 1조: 5점, 1000억~1조: 4점, 100~1000억: 3점, 100억 이하: 2점 |
| **구현 가능성** | /5 | 현재 팀 스택으로 6개월 내 MVP: 5점, 1년: 4점, 2년: 3점, 불가: 1점 |
| **경쟁 우위** | /5 | AI/기술로 모방 어려움: 5점, 네트워크 효과: 4점, 브랜드/특허: 3점, 낮음: 2점 |
| **수익 모델** | /5 | SaaS 반복 수익: 5점, 거래 수수료: 4점, 광고: 3점, 불명확: 1점 |
| **창업자 적합성** | /5 | 도메인 전문성 + 네트워크 보유: 5점, 일부 보유: 3점, 없음: 1점 |

**출력 형식**:
```markdown
| 아이디어 | 시장 | 구현 | 경쟁우위 | 수익 | 적합성 | 합계 | 추천 |
|---------|------|------|---------|------|--------|------|------|
| 아이디어A | 5 | 4 | 4 | 5 | 5 | 23 | ★ |
```

### 2.3 TOP5 상세 분석 → `02-ideas/idea-0{N}-{name}.md` (5개 파일)

각 아이디어별 상세 분석 문서:

```markdown
# {아이디어명}

## 한 줄 요약
## 해결하는 문제 (Problem)
## 솔루션 (Solution)
## 타겟 고객 (Customer Segments)
## 시장 규모 (TAM/SAM/SOM)
## 수익 모델 (Revenue Model)
## 경쟁 우위 (Unfair Advantage)
## MVP 핵심 기능 (Must Have / Should Have / Nice to Have)
## 핵심 리스크 & 미티게이션
## 성공 시나리오 vs 피봇 시나리오
## 6개월 목표 지표
```

## Phase 3: 비즈니스 모델 캔버스 (TOP3)

TOP3 아이디어에 대해 **BMC 9블록** 전체 작성 → `03-business-models/bmc-0{N}-{name}.md`

### BMC 9블록 구조

```markdown
# {서비스명} - 비즈니스 모델 캔버스

## 1. 고객 세그먼트 (Customer Segments)
| 세그먼트 | 규모 | 특성 |
- 주 타겟, 부 타겟, 3차 타겟 명시

## 2. 가치 제안 (Value Propositions)
핵심 메시지: "{한 줄 가치 제안}"
| 가치 | 기존 방식 | 새 방식 | 효과 |

## 3. 채널 (Channels)
- 획득 채널 (고객 유입)
- 유지 채널 (고객 리텐션)

## 4. 고객 관계 (Customer Relationships)
| 단계 | 방식 |
- 온보딩 / 일반 지원 / VIP / 커뮤니티

## 5. 수익 흐름 (Revenue Streams)
| 수익원 | 단가 | 비중 |
- 구독/수수료/일회성 등 명시
- 연간 구독 할인 정책

## 6. 핵심 자원 (Key Resources)
| 유형 | 자원 | 중요도 |
- 기술/인력/데이터/파트너십

## 7. 핵심 활동 (Key Activities)
- 제품 개발 / 마케팅 / 고객 성공

## 8. 핵심 파트너십 (Key Partnerships)
| 파트너 | 역할 | 중요도 |

## 9. 비용 구조 (Cost Structure)
| 항목 | 월 비용 | 연간 비용 |
| **총 합계** | X,XXX만원 | XX억원 |

## 재무 예측 (3년)
| 지표 | Year 1 | Year 2 | Year 3 |
- 유료 고객 수 / ARPU / MRR / ARR / 영업이익률

## BEP (손익분기점)
- BEP 조건 / 예상 시점 / 월 고정비

## 투자 계획
| 라운드 | 시기 | 금액 | 사용처 | 조건 |
- Seed → Seed+ → Series A
```

## 실행 원칙

### 병렬화 전략

```
Phase 1: 4개 리서치 병렬 실행 (ultrawork 활성화)
Phase 2: 아이디어 도출 (순차) → 평가 (순차) → TOP5 상세분석 병렬
Phase 3: TOP3 BMC 병렬 생성
Phase 4-6: 순차 실행
```

### 품질 기준

각 Phase 완료 기준:
- Phase 1: 4개 리서치 문서 모두 작성, 각 최소 500자 이상
- Phase 2: 평가 매트릭스 완성 + TOP5 상세 문서 각 완성
- Phase 3: BMC TOP3 각 9블록 모두 작성 + 재무 예측 포함
- Phase 4: 24개월 월별 계획 + 팀 로드맵 + 투자 계획 포함
- Phase 5: 이번주/월/3개월/6개월 체크리스트 + 핵심 가설 검증 계획 포함

### 사용자 확인 포인트

Phase 완료 시마다 사용자에게 진행 상황을 보고하고 계속 진행 여부 확인:
- Phase 1 완료 후: "4개 리서치 문서 완성. TOP5 아이디어 발굴로 진행할까요?"
- Phase 2 완료 후: "평가 매트릭스 완성. TOP3 추천: {A}, {B}, {C}. BMC 생성으로 진행할까요?"
- Phase 3 완료 후: "BMC 완성. 실행 로드맵 작성으로 진행할까요?"

### 데이터 부족 시 처리

웹 검색 불가 환경에서도 동작하도록:
- 학습 데이터 기반 분석 활용
- 명확하게 "추정치" 또는 "일반적 벤치마크" 라고 표기
- 실제 검증이 필요한 항목은 `[검증 필요]` 태그 사용
